# ⚡ Quick Start: Internal Release Lightning CLI

**🔒 INTERNAL RELEASE ONLY** - Not for public distribution

## What You Need to Do

You now have **everything needed** to package Lightning for internal organizational use.

---

## Step 1: Update Version (5 min)

Edit `package.json`:
```json
{
  "version": "1.0.0",
  "name": "@github/lightning",
  "description": "⚡ Lightning CLI: SLM-powered Copilot replacement"
}
```

Edit `Dockerfile`:
```dockerfile
ENV VERSION=1.0.0
```

Edit `homebrew-lightning.rb`:
```ruby
version "1.0.0"
```

---

## Step 2: Create Git Tag (2 min)

```bash
# Commit version changes
git add package.json Dockerfile homebrew-lightning.rb
git commit -m "IOT-123 Bump to v1.0.0 for release"

# Tag it
git tag v1.0.0
git push origin v1.0.0
```

**That's it!** GitHub Actions handles the rest.

---

## Step 3: Wait for CI/CD (~30 min)

GitHub Actions automatically:
- ✅ Builds all binaries (6 platforms)
- ✅ Runs tests + security check
- ✅ Generates checksums
- ✅ Creates Docker image
- ✅ Publishes to npm
- ✅ Uploads to GitHub Releases
- ✅ Publishes Docker image

**Status**: Watch https://github.com/github/lightning/actions

---

## Step 4: Verify Release (5 min)

```bash
# Test the installer works
bash install.sh

# Check npm package
npm info @github/lightning

# Check Docker image
docker pull github/lightning:v1.0.0

# Verify Homebrew (next day after PR approval)
brew install lightning-cli
```

## Installation Methods for Users

After release, users can install via:

```bash
# Method 1: Bash (Linux/macOS)
curl -fsSL https://lightning.dev/install | bash

# Method 2: npm (All platforms)
npm install -g @github/lightning

# Method 3: Homebrew (macOS/Linux)
brew install lightning-cli

# Method 4: Docker
docker pull github/lightning:v1.0.0

# Method 5: Windows (once WinGet approves)
winget install GitHub.Lightning
```

## Troubleshooting

### Release failed?
- Check GitHub Actions logs: Actions tab
- Common issues: Missing secrets (NPM_TOKEN, Docker credentials)

### Installer not working?
- Test locally: `bash install.sh`
- Verify checksums: `sha256sum -c checksums.sha256`

### npm publish stuck?
- Check authentication: `npm whoami`
- Check token: `cat ~/.npmrc`

### Docker image not pushing?
- Verify Docker credentials
- Check Docker Hub permissions
